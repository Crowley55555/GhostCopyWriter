# 🎫 Ручной генератор DEMO токенов для бета-теста

## Production (Docker на сервере)

Сайт доступен по **HTTPS на порту 443** (`https://IP/`, самоподписанный SSL). В `.env` на сервере: `SITE_URL=https://85.208.86.148`.

```bash
cd /opt/GhostCopyWriter

# Быстрая ссылка DEMO
docker compose -f docker-compose.production.yml exec django \
  python manual_token_generator.py --quick DEMO_FREE

# Токен разработчика
docker compose -f docker-compose.production.yml exec django \
  python manual_token_generator.py --quick DEVELOPER

# Интерактивное меню
docker compose -f docker-compose.production.yml exec -it django \
  python manual_token_generator.py
```

Явный URL (если `SITE_URL` в контейнере не задан):

```bash
docker compose -f docker-compose.production.yml exec django \
  python manual_token_generator.py --site-url https://85.208.86.148 --quick DEMO_FREE
```

Ссылки для пользователей: `https://85.208.86.148/auth/token/<uuid>/` (не `http://` и не порт 8010).

## Описание

Простой скрипт для ручной генерации токенов доступа во время бета-тестирования Ghostwriter.

**Особенности:**
- ✅ Запускается из PyCharm или терминала
- ✅ Генерация без проверок на дубликаты
- ✅ Массовая генерация токенов
- ✅ Экспорт ссылок в файл
- ✅ Статистика и управление токенами

## Быстрый старт

### 1. Запуск из PyCharm

1. Откройте файл `manual_token_generator.py` в PyCharm
2. Нажмите правой кнопкой → "Run 'manual_token_generator'"
3. Или используйте `Shift + F10`

### 2. Запуск из терминала

```bash
# Активируйте виртуальное окружение (если нужно)
venv\Scripts\activate  # Windows
# или
source venv/bin/activate  # Linux/macOS

# Запустите генератор
python manual_token_generator.py
```

## Режимы работы

### Интерактивный режим (по умолчанию)

```bash
python manual_token_generator.py
```

**Меню:**
- `1` - Сгенерировать DEMO токен (14 дней, безлимит)
- `2` - Сгенерировать MONTHLY токен (30 дней, безлимит)
- `3` - Сгенерировать YEARLY токен (365 дней, безлимит)
- `4` - Сгенерировать DEVELOPER токен (бессрочный, безлимит)
- `5` - Массовая генерация токенов
- `6` - Список активных токенов
- `7` - Статистика токенов
- `8` - Деактивировать токен
- `9` - Настройки генерации
- `0` - Выход

### Быстрая генерация (для скриптов)

```bash
# Генерирует один DEMO токен и выводит только ссылку
python manual_token_generator.py --quick
```

Пример использования в скрипте:
```bash
#!/bin/bash
# Генерируем 5 токенов
for i in {1..5}; do
    python manual_token_generator.py --quick >> demo_tokens.txt
done
```

## Настройки генерации

### Изменение параметров DEMO токенов

1. Запустите генератор
2. Выберите `9` (Настройки)
3. Введите новые значения:
   - Срок действия в днях (по умолчанию: 7)

**Пример:**
```
⚙️ Настройки генерации DEMO токенов

ℹ️ DEMO токены теперь без лимита генераций (только срок действия)

Срок действия в днях (текущее: 7): 14

✅ Настройки обновлены:
   Срок: 14 дней
   Лимит: Безлимит
```

## Типы токенов

### DEMO
- **Срок:** 14 дней (настраивается)
- **Лимит:** Безлимит
- **Использование:** Бесплатное тестирование

### MONTHLY
- **Срок:** 30 дней
- **Лимит:** Безлимит
- **Использование:** Расширенное тестирование

### YEARLY
- **Срок:** 365 дней
- **Лимит:** Безлимит
- **Использование:** Долгосрочное тестирование

### DEVELOPER
- **Срок:** Бессрочный (100 лет)
- **Лимит:** Безлимит
- **Использование:** Разработка и администрирование

## Управление токенами

### Деактивация токена

Если нужно отозвать доступ:

1. Запустите генератор
2. Выберите `8` (Деактивировать токен)
3. Введите UUID токена
4. Подтвердите

### Просмотр статистики

1. Запустите генератор
2. Выберите `7` (Статистика токенов)

**Результат:**
```
📈 Статистика токенов

Всего токенов: 50
Активных токенов: 32

По типам:
  DEMO: 25
  MONTHLY: 5
  YEARLY: 2
  DEVELOPER: 0
```

## Экспорт токенов

### Автоматическое сохранение в файл

При массовой генерации токены можно сохранить в текстовый файл:

**Формат файла:**
```
Ghostwriter - DEMO токены
Создано: 12.01.2026 14:30
Количество: 10
======================================================================

1. http://localhost:8000/auth/token/uuid-1/
2. http://localhost:8000/auth/token/uuid-2/
...
```

**Имя файла:** `tokens_<ТИП>_<ДАТА>_<ВРЕМЯ>.txt`

Пример: `tokens_DEMO_20260112_143025.txt`

## Автокопирование в буфер обмена

При генерации одного токена ссылка автоматически копируется в буфер обмена (если установлен `pyperclip`).

**Установка pyperclip (опционально):**
```bash
pip install pyperclip
```

## Безопасность

⚠️ **Важно:**
- Не коммитьте файлы с токенами в Git (они в `.gitignore`)
- Не публикуйте ссылки с токенами публично
- Деактивируйте токены после окончания бета-теста
- Храните файлы с токенами в безопасном месте

## Мониторинг бета-теста

Регулярно проверяйте:
- Список активных токенов (опция 6)
- Статистику использования (опция 7)
- Какие токены используются активно

**Полезная информация:**
- Количество генераций через токен (`total_used`)
- Последнее использование (`last_used`)
- IP адрес последнего использования (`current_ip`)
